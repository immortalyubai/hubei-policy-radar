## 数据链路

```text
管理员扫码登录
  -> 本机每 2 小时拉取 3 个公众号的新文章元数据
  -> 48 小时重叠窗口 + 每账号独立游标
  -> 标题/摘要政策关键词筛选
  -> 白名单校验 + 严格去重
  -> 政策雷达公开快照
  -> 后续由政府官网同源信息升级为“官网已核验”
```

默认不批量抓取公众号全文和图片。只有标题或摘要命中政策、申报、赛事、人工智能或 OPC 规则后，才进入待核验政策流。

## 本地运行

开发副本保存在已被 Git 忽略的 `work/wechat-download-api/`。为避免 macOS 后台任务无法读取“文稿”目录，常驻运行副本安装在 `~/Library/Application Support/PolicyRadar/wechat-download-api/`。登录凭证保存在运行副本的 `.env` 与 `data/.credentials.json`，不得提交。

启动后访问：

- 登录页：`http://127.0.0.1:5000/login.html`
- 订阅状态：`http://127.0.0.1:5000/rss.html`
- 健康检查：`http://127.0.0.1:5000/api/health`

政策雷达同步：

```bash
npm run collect:wechat-feed
node scripts/merge-static-data.mjs --input work/wechat-collection.json
npm run test:pages
```

采集配置位于 `config/wechat-sources.json`；每账号游标位于 `work/wechat-feed-state.json`。首次运行默认只回看最近 48 小时，整批成功后才推进游标。

## 安全边界

- 扫描服务只绑定本机回环地址，禁止直接暴露公网。
- 网站和公开 JSON 不保存或发布 Cookie、Token、二维码、登录账号 ID 或原始会话；用于白名单的公众号 FakeID 只留在采集配置，不进入网页数据。
- 公众号仅承担“提前发现”；政策效力、金额、截止日期最终以政府官网为准。
- 登录通常约 4 天到期；页面只公开预计到期时间，到期后必须由管理员重新扫码。
- 电脑休眠、程序停止或登录到期时，公众号扫描会暂停；政府官网定时采集不受影响。
- 上游 v1.6.0 的可选 MCP 依赖存在版本锁定兼容问题，本地试点保持 `ENABLE_MCP=0`，不启用或暴露 MCP。

## 静态网站边界

当前公开站点读取构建时生成的安全快照，不会直接访问本机 `127.0.0.1`。因此一次完整发布必须包含：本机扫描、规则筛选、快照合并、静态测试和站点部署。页面明确显示“状态快照”，不会把计划扫描冒充为实时在线状态。
