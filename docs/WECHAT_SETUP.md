# 微信公众号接入说明

## 已完成：公开单篇文章

部署后，`POST /api/import/wechat` 接受一个公共文章 URL。接口校验域名与路径，使用固定公共解析服务，清洗为纯文本并调用统一入库流程。

安全规则：

- 不接受账号密码、Cookie、Token 或 `X-Auth-Key`。
- 不保存全量原始 JSON、`sessionid`、评论、点赞、奖励或用户状态。
- 未知公众号自动进入 `quarantined`，不会出现在正式政策流。
- 白名单公众号仍只是“优先发现”，官网证据才是最终核验依据。

## 待用户动作：账号级定时扫描

“自动发现某个公众号所有新文章”依赖账号级文章列表，不是单篇公共链接能力。启用时由用户本人访问 [down.mptext.top](https://down.mptext.top) 扫码登录，再在其界面选择需要监控的公众号。

项目只接收扫码后导出的公开文章结果，不接触或保存微信会话凭证。建议从 10–30 个湖北高价值官方公众号白名单开始，验证两周后再扩容。

首批建议白名单：湖北发布、湖北科技、武汉科技创新、武汉经信、光谷、湖北人社、湖北省经信厅、各地市科技与经信主管部门。

## 扫描频率建议

- 核心公众号：每 2 小时。
- 普通地方账号：每天 3–4 次。
- 发现新文章后立即入库为 `pending_official`。
- 同标题官网记录出现后追加来源证据并升级为 `official_verified`。
